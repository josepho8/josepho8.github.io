Key Findings

What the data revealed

Six variables were analysed for their impact on economy fares. Booking window had the largest single effect — followed by season, then day of week.

Optimal Booking Window
22–45 days
The sweet spot before departure. Prices are ~51% lower than last-minute and ~14% lower than booking 3+ months ahead.
Cheapest Day to Fly
Tue or Wed
Tuesday and Wednesday departures are consistently cheapest. Friday and Sunday are most expensive — up to 18% above the weekly average.
Cheapest Season
Sep – Oct
Fall travel is 10–15% cheaper than summer peak (Jul–Aug). December holiday travel rivals summer pricing.
Cheapest Departure Time
6 – 8 AM
Early morning departures tend to carry lower fares than midday or evening flights on the same routes.
Nonstop vs. Connecting
~12% saved
Accepting one connection saves an average of 12% on economy. The premium is highest on international routes.
Price Volatility
High
Fare ranges exceed $800 on long-haul routes (YYZ→LHR, YVR→NRT), making timing far more impactful than on short-hauls.

Query 1.1 — Booking window bucketing with CASE WHEN
-- Average economy fare by booking window bucket
SELECT
    CASE
        WHEN days_until_flight BETWEEN 0  AND 7   THEN 'Last-minute (0–7 days)'
        WHEN days_until_flight BETWEEN 8  AND 21  THEN '1–3 weeks (8–21 days)'
        WHEN days_until_flight BETWEEN 22 AND 45  THEN '3–6 weeks (22–45 days)'
        WHEN days_until_flight BETWEEN 46 AND 90  THEN '1.5–3 months (46–90 days)'
        ELSE                                        '3+ months (91+ days)'
    END                           AS booking_window,
    COUNT(*)                      AS num_flights,
    ROUND(AVG(base_fare), 2)      AS avg_fare,
    ROUND(MIN(base_fare), 2)      AS min_fare,
    ROUND(MAX(base_fare), 2)      AS max_fare
FROM flights
WHERE cabin_class = 'economy'
GROUP BY booking_window
ORDER BY booking_window;
Query 1.4 — Savings vs. last-minute using CTE
-- How much do you save vs. last-minute? Calculated via CTE self-reference.
WITH window_avgs AS (
    SELECT
        CASE
            WHEN days_until_flight BETWEEN 0  AND 7  THEN '0–7 days'
            WHEN days_until_flight BETWEEN 8  AND 21 THEN '8–21 days'
            WHEN days_until_flight BETWEEN 22 AND 45 THEN '22–45 days'
            WHEN days_until_flight BETWEEN 46 AND 90 THEN '46–90 days'
            ELSE '91+ days'
        END AS booking_window,
        ROUND(AVG(base_fare), 2) AS avg_fare
    FROM flights
    WHERE cabin_class = 'economy'
    GROUP BY booking_window
),
last_min AS (
    SELECT avg_fare AS lm_fare FROM window_avgs WHERE booking_window = '0–7 days'
)
SELECT
    w.booking_window,
    w.avg_fare,
    ROUND(lm.lm_fare - w.avg_fare, 2)                        AS savings_vs_last_minute,
    ROUND(100.0 * (lm.lm_fare - w.avg_fare) / lm.lm_fare, 1) AS pct_cheaper
FROM window_avgs w, last_min lm
ORDER BY w.booking_window;
Section 7

Advanced window functions

Window functions were used to rank prices within route segments, compute rolling averages, and flag deals in the bottom 20th percentile.

Query 7.2 — Price percentile rank with PERCENT_RANK()
-- Flag every record as a "Great deal", "Good price", "Average", or "Expensive"
-- based on its percentile rank within the same route and cabin class.
SELECT
    flight_id,
    origin || ' → ' || destination        AS route,
    cabin_class,
    airline_name,
    base_fare,
    ROUND(PERCENT_RANK() OVER (
        PARTITION BY origin, destination, cabin_class
        ORDER BY base_fare
    ) * 100, 1)                          AS price_percentile,
    CASE
        WHEN PERCENT_RANK() OVER (
            PARTITION BY origin, destination, cabin_class
            ORDER BY base_fare) <= 0.20 THEN 'Great deal'
        WHEN PERCENT_RANK() OVER (
            PARTITION BY origin, destination, cabin_class
            ORDER BY base_fare) <= 0.40 THEN 'Good price'
        WHEN PERCENT_RANK() OVER (
            PARTITION BY origin, destination, cabin_class
            ORDER BY base_fare) <= 0.70 THEN 'Average'
        ELSE                                         'Expensive'
    END                                          AS price_label
FROM flights
ORDER BY price_percentile;
Query 7.1 — Rolling 7-day average with ROWS BETWEEN
-- Smooth out daily price noise with a 7-day rolling average per route.
WITH daily_avg AS (
    SELECT
        origin || ' → ' || destination AS route,
        search_date,
        ROUND(AVG(base_fare), 2)       AS daily_avg_fare
    FROM flights
    WHERE cabin_class = 'economy'
    GROUP BY route, search_date
)
SELECT
    route,
    search_date,
    daily_avg_fare,
    ROUND(AVG(daily_avg_fare) OVER (
        PARTITION BY route
        ORDER BY search_date
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ), 2)                          AS rolling_7day_avg
FROM daily_avg
ORDER BY route, search_date;
Technical Skills

SQL concepts demonstrated

The project covers 9 analytical sections and ~60 queries progressively increasing in complexity.

CTEs (WITH clauses)
Multi-step calculations broken into readable named subqueries — used for savings comparisons and route ranking.
Window Functions
RANK(), PERCENT_RANK(), ROW_NUMBER(), FIRST_VALUE() and rolling averages with ROWS BETWEEN.
Conditional Aggregation
AVG(CASE WHEN ... END) used to pivot nonstop vs. connecting fares side-by-side in a single query.
CASE WHEN Bucketing
Custom booking window segments, time-of-day groups, and seasonal categories built inline without temp tables.
Data Quality Checks
Null audits, row count validation, and distribution checks run before any analysis queries.
UNION ALL Summaries
Executive summary view combining all key findings into a single structured output table.
